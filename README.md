> 本仓库内容仅供学习交流使用，请不要依赖自动签到，确保自己提交的信息真实有效。参与抗疫人人有责，请在使用程序前仔细阅读代码注释和本文档，请不要将此程序用于学习交流以外的任何用途，否则由此引发的一切后果由使用者本人承担。




自打去年四月起就有这么一个想法------干掉校园防疫签到。奈何多次尝试抓包无果，只能退而求其次，一直使用Auto.js的自动点击脚本实现定时签到。弊端很明显，Auto.js的运行需要基于Android环境，并且要保证程序常驻在后台，因此经常会漏签。前两天翻GitHub的时候无意看到了一个开源项目中有抓包成功并且提供了API，项目中的PHP程序在Linux环境下运行好像有点问题，访问签到API后无法获得服务器返回的内容进而报json解析错误。本人PHP很菜，仅仅局限于能做点CTF代码审计的水平，能看懂代码，排查问题却无从下手，简单调试了一会儿便选择放弃。为了方便部署在Linux环境下，便抽空写了个Python版本。




本项目基于<a href='https://github.com/PrintNow'>PrintNow(Chuwen)</a>大佬的<a href='https://github.com/PrintNow/Jiangxi-University-Health-Check-in'>Jiangxi-University-Health-Check-in</a>项目的抓包结果，由本人独立开发出Python版本，功能性并无差异，使用requests库的session技术跨请求保留cookie，达到简化程序的目的。可以在Windows中设定计划任务定时执行，也可在Linux使用crontab定时执行。有开发能力的朋友可以考虑绑定一个微信机器人，实时推送签到情况，因为我个人使用的是一个小众的机器人，不方便推荐，便删除了发布版本中的相关代码。




程序整体结构很简单，主要分为三部分

* 配置用户信息（用户需要改动的部分）
* 使用学校代码和用户学号请求登录页面，获取cookie
* 使用cookie请求签到页面，post提交用户的配置信息，签到成功，返回签到状态




关于配置信息的细节，代码中有很详细的注释，此处不再赘述。

关于抓包过程，<a href='https://github.com/PrintNow/Jiangxi-University-Health-Check-in'>Jiangxi-University-Health-Check-in</a>项目中有详细的分析，此处不再赘述。

代码可以在Windows和Linux的Python3环境下正常运行，不过也没有经过很严谨的测试，只是抱着写着玩玩的心态（毕竟马上就开学了orz），如果在使用中发现有问题，欢迎在项目<a href='https://github.com/JetXi/JiangXi-University-Antiepidemic-Clock-In/issues'>issues</a>反馈，也可发送邮件至我的邮箱。
